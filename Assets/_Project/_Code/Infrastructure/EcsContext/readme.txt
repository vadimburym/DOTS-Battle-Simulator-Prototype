EcsContext - обособленный контекст ECS систем.
Когда вы пишите систему - она автоматически создается и начинает тикать после запуска игры.
Чтобы контроллировать этот процесс вручную, нужно для системы поставить аттрибут
[DisableAutoCreation] и создавать вручную в вашей группе и управлять включением/отключением группы.

системы могут быть 2 типов:
1) Managed: class SystemBase, управляемая память, GC, - в них можно [Inject] любых managed-сервисов, но они медленнее
чем unmanaged (хотя во всех ECS-фрейморвках, например в Leo unmanaged систем в принципе нет, они все Managed).
Рекомендуется их использовать когда это не горячая точка и нужна максимальная гибкость, т.к. в них
скорее всего не получится сделать Jobs, Burst.

2) Unmanaged: struct ISystem, неуправляемая память - в них нельзя [Inject], нельзя получать любые managed-зависимости,
использовать любые ссылки на классы и тд. Работает только с плоскими данными. Легко Jobs, Burst.
Рекомендуется их использовать везде где возможно, особенно в горячих точках и когда нужна максимальная производительность.

EcsContext - обертка над определенной группой систем и EntityManager'ом. Этот контекст можно прогреть нужными системами
через удобный EcsContextBuilder, и делать их Enable/Disable
Сборка производится через Installer.

Бывает такое, что редактор подсвечивает всю систему красным, скорее всего это баг из-за [DisableAutoCreation]. Нужно сделать
любое изменение и убрать его, чтобы ошибка исчезла
